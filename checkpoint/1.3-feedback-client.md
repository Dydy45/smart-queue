## 📋 Vue d'ensemble

Implémenter un système de feedback client permettant aux utilisateurs de noter leur expérience après avoir été servis. Cela permet aux entreprises de mesurer la satisfaction client, d'identifier les points d'amélioration et de valoriser les services de qualité.

### Bénéfices attendus
- ✅ **Mesure de satisfaction** : Indicateur quantitatif de la qualité du service
- ✅ **Amélioration continue** : Identification des points faibles via les commentaires
- ✅ **Valorisation du personnel** : Reconnaissance des agents performants
- ✅ **Transparence** : Les clients voient que leur avis compte
- ✅ **Analytics** : Données exploitables pour le dashboard (moyenne, tendances)
- ✅ **Engagement** : Les clients se sentent écoutés et impliqués

---

## 🏗️ Architecture Technique

### Stack
- **Prisma** : Modèle `Feedback` avec relation vers `Ticket` et `Company`
- **Server Actions** : `submitFeedback`, `getFeedbackStats`, `getRecentFeedbacks`
- **UI Components** : Composant `FeedbackModal` avec étoiles interactives + textarea
- **Validation** : Zod schema pour rating (1-5) et commentaire (optionnel, max 500 chars)

### Flux utilisateur

```
1. Client termine son service (ticket FINISHED)
   ↓
2. Modal de feedback s'affiche automatiquement (ou lien dans la page ticket)
   ↓
3. Client sélectionne 1-5 étoiles + commentaire optionnel
   ↓
4. Soumission → Enregistrement en BDD avec ticketId, companyId, rating, comment
   ↓
5. Confirmation visuelle + fermeture du modal
   ↓
6. OWNER/ADMIN voit les feedbacks dans le dashboard (moyenne, liste, filtres)
```

### Modèle de données

```prisma
model Feedback {
  id          String   @id @default(cuid())
  ticketId    String   @unique
  ticket      Ticket   @relation(fields: [ticketId], references: [id], onDelete: Cascade)
  companyId   String
  company     Company  @relation(fields: [companyId], references: [id], onDelete: Cascade)
  rating      Int      // 1-5 étoiles
  comment     String?  // Commentaire optionnel (max 500 chars)
  createdAt   DateTime @default(now())
  
  @@index([companyId])
  @@index([createdAt])
}
```

## 🔐 Sécurité & Bonnes pratiques

### Validation stricte
- ✅ **Rating** : Obligatoire, entre 1 et 5
- ✅ **Commentaire** : Optionnel, max 500 caractères
- ✅ **Unicité** : Un seul feedback par ticket (contrainte unique sur ticketId)

### Permissions
- ✅ **Soumission** : Tout le monde peut soumettre un feedback pour un ticket FINISHED
- ✅ **Consultation** : Seuls OWNER/ADMIN peuvent voir les feedbacks de leur entreprise
- ✅ **Suppression** : Pas de suppression (audit trail)

### Anti-spam
- ✅ **Rate limiting** : Max 5 feedbacks par IP par heure
- ✅ **Validation ticket** : Le ticket doit exister et être FINISHED
- ✅ **Pas de doublons** : Contrainte unique sur ticketId
