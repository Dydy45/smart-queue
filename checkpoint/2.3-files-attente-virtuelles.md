**Objectif** : Permettre au client de prendre un ticket depuis chez lui et recevoir des notifications basées sur sa géolocalisation pour savoir quand se déplacer

## 📋 Vue d'ensemble

### Problème actuel

Le client doit être **physiquement présent** pour prendre un ticket et attendre sur place. Cela génère :
- Des salles d'attente bondées
- Du temps perdu pour le client
- Aucune visibilité sur le temps d'attente avant de se déplacer
- Pas d'incitation à venir au bon moment

### Solution : File d'attente virtuelle avec géolocalisation

```
Client chez lui                    Client en route                    Client sur place
      │                                  │                                  │
      ▼                                  ▼                                  ▼
Prend un ticket virtuel        GPS → mise à jour position         Notification "Vous êtes arrivé"
via /page/{pageName}           + estimation du trajet              Le ticket est traité normalement
      │                                  │
      ▼                                  ▼
Reçoit un lien de suivi        Notification "Partez maintenant !"
/track/{ticketNum}             quand le temps estimé ≈ temps trajet
```

### Ce qui existe déjà vs ce qu'on ajoute

| Fonctionnalité | Existe déjà ? | Action |
|----------------|---------------|--------|
| Prise de ticket via page publique | ✅ Oui (`/page/{pageName}`) | Ajouter option "ticket virtuel" |
| Suivi du ticket (polling 5s) | ✅ Oui (page client) | Créer page de tracking dédiée |
| Notifications WhatsApp | ✅ Oui (quand tour approche) | Ajouter notification de départ + proximité |
| Estimation ML du temps d'attente | ✅ Oui (EWMA) | Exploiter pour calculer quand partir |
| Géolocalisation client | ❌ Non | Ajouter (Geolocation API browser) |
| Localisation entreprise | ❌ Non | Ajouter coordonnées dans Company |
| Page de tracking temps réel | ❌ Non | Créer `/track/{ticketNum}` |

## 🔐 Sécurité & Vie privée

### Sécurité
- ✅ Le `trackingToken` est un UUID (non-devinable), différent du `ticketNum`
- ✅ La position GPS est stockée temporairement et supprimée quand le ticket est FINISHED
- ✅ Les coordonnées de l'entreprise sont publiques (normal pour un commerce)
- ✅ Rate limiting sur `updateClientLocation()` (max 1 appel / 10s par token)
- ✅ Validation des coordonnées GPS (lat: -90 à 90, lng: -180 à 180)
- ✅ La page `/track/{token}` est publique mais le token est non prédictible

### Vie privée (RGPD)
- ✅ Consentement explicite avant collecte de géolocalisation
- ✅ Les données GPS sont supprimées automatiquement quand le ticket est terminé (FINISHED)
- ✅ Le client peut désactiver le GPS à tout moment (fallback : pas de notification proximité)
- ✅ Aucun historique de positions conservé (seule la dernière position est stockée)

### Performance
- ✅ Calcul Haversine côté serveur (< 1ms)
- ✅ Pas d'API de géolocalisation tierce (tout en browser natif + calcul serveur)
- ✅ Polling de position toutes les 30s (pas de WebSocket, cohérent avec l'archi existante)
- ✅ Batch des vérifications de notification lors des changements de file

## 🆚 Comparaison : Avant vs Après

| Aspect | Avant (sur place) | Après (virtuel) |
|--------|-------------------|------------------|
| **Lieu d'attente** | Salle d'attente physique | Chez soi / en route |
| **Visibilité** | Position dans la file (polling) | Position + distance + ETA + carte |
| **Notification** | WhatsApp "votre tour approche" | + "Partez maintenant" + "Vous êtes arrivé" |
| **Temps perdu** | Attente complète sur place | Déplacement synchronisé avec la file |
| **Capacité salle** | Limitée physiquement | Files plus longues sans encombrement |
