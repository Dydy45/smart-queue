## 📋 Vue d'ensemble

Implémenter un système complet de prise de rendez-vous en ligne permettant aux clients de réserver un créneau horaire pour un service spécifique. Le système inclut un calendrier interactif, la gestion des disponibilités, la confirmation par email/SMS/WhatsApp, et des rappels automatiques avant le rendez-vous.

### Bénéfices attendus
- ✅ **Réduction de l'attente** : Les clients viennent à l'heure prévue, pas de file d'attente
- ✅ **Optimisation du planning** : Meilleure répartition de la charge de travail
- ✅ **Expérience client améliorée** : Planification à l'avance, moins de stress
- ✅ **Réduction du no-show** : Rappels automatiques diminuent les absences
- ✅ **Professionnalisme** : Image moderne et organisée de l'entreprise
- ✅ **Données exploitables** : Statistiques sur les créneaux les plus demandés

## 🏗️ Architecture Technique

### Stack
- **Prisma ORM** : Modèles `Appointment`, `TimeSlot`, `AppointmentReminder`
- **Zod** : Validation des données de rendez-vous
- **Next.js Server Actions** : CRUD rendez-vous, gestion disponibilités
- **React Client Components** : Calendrier interactif, sélection créneaux
- **Cron Jobs / Background Tasks** : Envoi des rappels automatiques
- **Notifications** : Email (Resend), SMS (Twilio), WhatsApp

### Modèle de données

```prisma
model Appointment {
  id              String   @id @default(cuid())
  companyId       String
  company         Company  @relation(fields: [companyId], references: [id], onDelete: Cascade)
  serviceId       String
  service         Service  @relation(fields: [serviceId], references: [id], onDelete: Cascade)
  postId          String?
  post            Post?    @relation(fields: [postId], references: [id], onDelete: SetNull)
  
  // Informations client
  clientName      String
  clientEmail     String?
  clientPhone     String?
  
  // Date et heure
  appointmentDate DateTime
  duration        Int      // en minutes
  
  // Statut
  status          AppointmentStatus @default(PENDING)
  
  // Métadonnées
  notes           String?  @db.Text
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  cancelledAt     DateTime?
  cancelReason    String?
  
  // Relations
  reminders       AppointmentReminder[]
  
  @@index([companyId])
  @@index([serviceId])
  @@index([appointmentDate])
  @@index([status])
}

enum AppointmentStatus {
  PENDING       // En attente de confirmation
  CONFIRMED     // Confirmé
  CANCELLED     // Annulé
  COMPLETED     // Terminé
  NO_SHOW       // Client absent
}

model AppointmentReminder {
  id              String   @id @default(cuid())
  appointmentId   String
  appointment     Appointment @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  
  reminderType    ReminderType
  sentAt          DateTime?
  scheduledFor    DateTime
  status          ReminderStatus @default(PENDING)
  
  @@index([appointmentId])
  @@index([scheduledFor])
  @@index([status])
}

enum ReminderType {
  EMAIL
  SMS
  WHATSAPP
}

enum ReminderStatus {
  PENDING
  SENT
  FAILED
}

model BusinessHours {
  id          String   @id @default(cuid())
  companyId   String
  company     Company  @relation(fields: [companyId], references: [id], onDelete: Cascade)
  
  dayOfWeek   Int      // 0 = Dimanche, 1 = Lundi, ..., 6 = Samedi
  openTime    String   // Format "HH:mm" (ex: "09:00")
  closeTime   String   // Format "HH:mm" (ex: "18:00")
  isOpen      Boolean  @default(true)
  
  @@unique([companyId, dayOfWeek])
  @@index([companyId])
}

model ClosedDate {
  id          String   @id @default(cuid())
  companyId   String
  company     Company  @relation(fields: [companyId], references: [id], onDelete: Cascade)
  
  date        DateTime @db.Date
  reason      String?
  
  @@index([companyId])
  @@index([date])
}
```

## 📝 Notes & Remarques

### Limitations connues
- **Pas de récurrence** : Impossible de créer des RDV récurrents (hebdomadaires, mensuels)
- **Un seul service par RDV** : Pas de RDV multi-services
- **Pas de liste d'attente** : Si un créneau est annulé, pas de notification automatique

### Optimisations futures
- **Récurrence** : RDV hebdomadaires/mensuels
- **Multi-services** : Réserver plusieurs services en un RDV
- **Liste d'attente** : Notification si un créneau se libère
- **Paiement en ligne** : Acompte ou paiement total lors de la réservation
- **Synchronisation calendrier** : Sync bidirectionnelle avec Google Calendar
- **IA de recommandation** : Suggérer les meilleurs créneaux selon l'historique

### Alternatives
- **Intégration Calendly** : Utiliser un service externe
- **Intégration Google Calendar API** : Synchronisation directe
- **Booking.com API** : Pour secteur hôtellerie/restauration
