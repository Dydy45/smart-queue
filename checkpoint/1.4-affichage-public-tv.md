## 📋 Vue d'ensemble

Créer une page d'affichage public optimisée pour être projetée sur un écran TV dans la salle d'attente. Cette page affiche en temps réel les tickets en cours d'appel, les tickets en attente, et permet aux clients de suivre visuellement l'avancement de la file sans avoir à consulter leur téléphone.

### Bénéfices attendus
- ✅ **Visibilité améliorée** : Les clients voient leur position dans la file sur grand écran
- ✅ **Réduction du stress** : Transparence totale sur l'état de la file d'attente
- ✅ **Professionnalisme** : Image moderne et organisée de l'entreprise
- ✅ **Accessibilité** : Information visible par tous, même sans smartphone
- ✅ **Réduction des questions** : Moins de sollicitations du personnel
- ✅ **Engagement** : Les clients restent informés et patients

## 🏗️ Architecture Technique

### Stack
- **Next.js App Router** : Route publique `/display/[pageName]`
- **React Server Components** : Rendu initial côté serveur
- **Polling/WebSocket** : Actualisation temps réel (toutes les 3-5 secondes)
- **TailwindCSS + DaisyUI** : Design responsive et moderne
- **Animations** : Transitions fluides pour les changements de statut

### Flux utilisateur

```
1. OWNER/ADMIN configure l'affichage public
   ↓
2. URL générée : /display/[pageName]
   ↓
3. Écran TV affiche la page en plein écran (mode kiosque)
   ↓
4. Actualisation automatique toutes les 3-5 secondes
   ↓
5. Affichage des tickets :
   - CALL : Tickets actuellement appelés (gros, en haut, animés)
   - IN_PROGRESS : Tickets en cours de traitement
   - PENDING : Prochains tickets en attente (top 5-10)
   ↓
6. Animations sonores/visuelles lors des appels (optionnel)
```

      // Polling toutes les 5 secondes

## 🔐 Sécurité & Bonnes pratiques

### Accès public
- ✅ **Route publique** : `/display/[pageName]` accessible sans authentification
- ✅ **Données limitées** : Afficher uniquement les numéros de tickets, pas les noms complets
- ✅ **Pas de modification** : Lecture seule, aucune action possible depuis l'écran

### Performance
- ✅ **Polling optimisé** : 5 secondes (compromis entre temps réel et charge serveur)
- ✅ **Limite de tickets** : Afficher max 10 tickets PENDING pour éviter le scroll
- ✅ **Cache** : Utiliser React Query ou SWR pour optimiser les requêtes

### UX
- ✅ **Animations fluides** : Transitions douces pour éviter les saccades
- ✅ **Lisibilité** : Tailles de police adaptées à la distance de lecture
- ✅ **Contraste** : Respect des normes WCAG AAA pour visibilité maximale

## 📝 Notes & Remarques

### Limitations connues
- **Pas de WebSocket** : Utilisation de polling simple (peut être amélioré en Phase 2)
- **Pas de son par défaut** : Nécessite interaction utilisateur pour activer l'audio
- **Pas de multi-écrans** : Un seul affichage par entreprise (peut être étendu)

### Optimisations futures
- **WebSocket** : Actualisation instantanée sans polling
- **Multi-écrans** : Affichage différent par service ou par étage
- **Personnalisation** : Couleurs, logo, messages personnalisés
- **Statistiques** : Temps d'attente moyen affiché en temps réel
- **QR Code** : Générer un QR code pour que les clients scannent et suivent leur ticket
- **Synthèse vocale** : Annonce vocale des numéros appelés

### Alternatives
- **Intégration écrans LED** : API pour écrans LED professionnels
- **Application mobile dédiée** : App pour tablette en mode kiosque
- **Chromecast** : Diffusion via Chromecast
