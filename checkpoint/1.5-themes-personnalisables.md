# 🎨 Checkpoint 1.5 — Thèmes Personnalisables

**Objectif** : Permettre aux entreprises de customiser logo, couleurs et branding sur leurs pages publiques  
**Impact** : Moyen  
**Effort** : Faible  
**Statut** : � Terminé

---

## 📋 Vue d'ensemble

Chaque entreprise peut personnaliser l'apparence de ses pages publiques (file d'attente, rendez-vous, affichage TV) avec ses propres couleurs, logo et nom affiché. Le thème est stocké en base et appliqué dynamiquement via CSS variables.

### Bénéfices attendus
- ✅ **Identité visuelle** : Les clients reconnaissent immédiatement l'entreprise
- ✅ **Professionnalisme** : Pages brandées au lieu de pages génériques
- ✅ **Différenciation** : Chaque entreprise a sa propre identité

---

## 🏗️ Architecture technique

### Modèle de données

Ajout de champs dans le modèle `Company` :

```prisma
model Company {
  // ... champs existants
  logoUrl       String?   // URL du logo (upload ou lien externe)
  primaryColor  String?   // Couleur principale (#hex)
  accentColor   String?   // Couleur d'accent (#hex)
  description   String?   // Description courte de l'entreprise
}
```

### Application du thème

Le thème est appliqué via des CSS custom properties (`--primary`, `--accent`) injectées dans un `<style>` tag sur les pages publiques. DaisyUI supporte nativement la personnalisation via `data-theme` et CSS variables.
