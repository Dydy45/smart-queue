## 📋 Vue d'ensemble

Intégrer WhatsApp Business API pour envoyer des notifications automatiques aux clients lorsque leur ticket est sur le point d'être appelé. WhatsApp offre un meilleur taux d'engagement, des coûts réduits et une expérience utilisateur enrichie par rapport aux SMS classiques.

### Bénéfices attendus
- ✅ **Coût réduit** : ~0.005€/message vs 0.08€ pour SMS (16x moins cher)
- ✅ **Taux d'ouverture supérieur** : ~98% vs 82% pour SMS
- ✅ **Expérience enrichie** : messages avec emojis, boutons interactifs, confirmation de lecture
- ✅ **Adoption massive** : quasi-universelle dans votre secteur
- ✅ **Réduction de l'anxiété** : clients peuvent s'éloigner de la salle d'attente
- ✅ **Différenciation concurrentielle** : communication moderne et professionnelle

## 🏗️ Architecture Technique

### Stack
- **Service WhatsApp** : Twilio WhatsApp Business API (ou Meta WhatsApp Cloud API)
- **Backend** : Next.js API Routes (Server Actions)
- **Base de données** : Prisma + PostgreSQL (ajout champ `phoneNumber` au modèle `Ticket`)
- **Déclencheur** : Webhook basé sur la position dans la file

### Flux de données
```
1. Client crée un ticket → saisit son numéro WhatsApp (optionnel avec opt-in)
2. Ticket stocké en BDD avec phoneNumber + whatsappConsent
3. Système surveille la position du ticket dans la file
4. Quand position <= 2 (2 tickets avant lui) → Déclenche notification WhatsApp
5. Twilio envoie le message via WhatsApp Business API (template pré-approuvé)
6. Log de l'envoi stocké en BDD (pour éviter les doublons)
7. Client reçoit le message avec bouton "Je suis là" (optionnel)
```

## 🔐 Sécurité & Conformité

### RGPD
- ✅ **Consentement explicite** : checkbox "J'accepte de recevoir des notifications WhatsApp pour ce ticket"
- ✅ **Base légale** : consentement (Art. 6.1.a RGPD) + intérêt légitime (amélioration du service)
- ✅ **Stockage minimal** : numéro de téléphone stocké uniquement pendant la durée du ticket
- ✅ **Droit à l'oubli** : suppression automatique du numéro après traitement du ticket (ou 30 jours max)
- ✅ **Transparence** : politique de confidentialité mise à jour avec mention WhatsApp
- ✅ **Portabilité** : export des données personnelles sur demande

### Règles WhatsApp Business
- ✅ **Opt-in obligatoire** : le client doit consentir explicitement avant le 1er message
- ✅ **Templates approuvés** : tous les messages doivent utiliser des templates validés par Meta
- ✅ **Pas de spam** : respecter la fenêtre de 24h et les règles d'usage
- ✅ **Opt-out facile** : le client peut se désinscrire à tout moment (commande STOP)

### Rate Limiting
- ✅ Limiter à 1 message WhatsApp par ticket (flag `whatsappNotified`)
- ✅ Éviter le spam : délai minimum de 5 minutes entre 2 messages pour le même numéro
- ✅ Quota quotidien : max 1000 messages/jour (limite Twilio sandbox, ajustable en production)

## 💰 Coûts estimés

### Twilio WhatsApp Pricing (indicatif)
- **Message WhatsApp sortant** : ~0.005€ par message (France)
- **Message WhatsApp entrant** : gratuit
- **Numéro WhatsApp Business** : ~1€/mois (ou gratuit avec sandbox)
- **Volume estimé** : 100 tickets/jour × 30 jours = 3000 messages/mois → **~15€/mois**

### Comparaison SMS vs WhatsApp
| Critère | SMS | WhatsApp | Économie |
|---------|-----|----------|----------|
| Coût unitaire | 0.08€ | 0.005€ | **94%** |
| 3000 messages/mois | 240€ | 15€ | **225€/mois** |
| 10 000 messages/mois | 800€ | 50€ | **750€/mois** |

### Optimisations possibles
- Utiliser le sandbox Twilio pour les tests (gratuit)
- Négocier un tarif entreprise avec Twilio pour volumes > 50k messages/mois
- Alternative : Meta WhatsApp Cloud API (encore moins cher, mais plus complexe à configurer)

## 📝 Notes & Remarques

### Alternatives à Twilio WhatsApp
- **Meta WhatsApp Cloud API** : Gratuit jusqu'à 1000 conversations/mois, puis ~0.003€/msg (encore moins cher)
- **Vonage WhatsApp** : Tarifs similaires à Twilio, interface différente
- **MessageBird** : Bonne alternative européenne

### Points d'attention
- **Délai d'approbation templates** : Prévoir 24-48h, parfois plus si rejet initial
- **Fenêtre de 24h** : Après le 1er message du client, vous pouvez répondre librement pendant 24h (utile pour support)
- **Vérification Business** : Pour un numéro dédié, prévoir 1-2 semaines de vérification Meta
- **Internationalisation** : Utiliser `libphonenumber-js` pour valider et formater les numéros selon le pays
- **Accessibilité** : Proposer aussi des notifications email en fallback pour ceux sans WhatsApp

### Évolutions futures (Phase 2)
- **Notifications push** : Pour l'app mobile native (quand développée)
- **Chatbot WhatsApp** : Répondre aux questions fréquentes automatiquement
- **Confirmation interactive** : Bouton "Je suis là" qui met à jour le statut du ticket
- **Rappels automatiques** : Si le client ne se présente pas 5 min après notification
